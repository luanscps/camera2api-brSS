01.

Sobre o App

🎯

O Projeto

Transforma um Samsung Galaxy Note10+ (ou qualquer Android com Camera2 API) em um servidor RTSP profissional. Stream de vídeo H.264 a 720p/30fps com controle remoto completo via painel web moderno. Latência de 200–500ms na rede local WiFi.

📡

Servidor RTSP

Utiliza a biblioteca RootEncoder v2.4.5 para transmissão de vídeo H.264 na porta 8554. Compatível com VLC, OBS Studio, ffplay e qualquer player RTSP.

🌐

Painel Web

Servidor HTTP via NanoHTTPD na porta 8080. Interface responsiva com controles dinâmicos que se adaptam automaticamente às capabilities de cada câmera detectada.

⚙️

Controles Manuais

Ajuste de ISO (50–3200), Exposição (1/8000s – 30s), Foco manual, Balanço de branco com múltiplos modos, zoom digital e flash/OIS quando disponíveis.

🔧

Stack Técnica

Desenvolvido em Kotlin com Coroutines para operações assíncronas. Serialização JSON com Gson 2.10.1. Target Android 7.0+ (API 24+).

02.

Funções Principais

🔍 discoverAllCameras()
Detecção Dinâmica de Câmeras
Percorre todas as câmeras do dispositivo via CameraManager, extraindo CameraCharacteristics: hardware level (LEGACY / LIMITED / FULL / LEVEL_3), ranges de ISO, exposição, foco, zoom, modos AF/AWB, presença de flash e OIS.
Camera2Controller.kt CameraCharacteristics INFO_SUPPORTED_HARDWARE_LEVEL
📊 /api/capabilities
Endpoint REST de Capabilities
Endpoint HTTP exposto pelo WebControlServer que serializa todas as capabilities detectadas em JSON e as retorna ao browser. Inclui header Access-Control-Allow-Origin: * para acesso cross-origin.
WebControlServer.kt NanoHTTPD Gson / JSON
🖥️ initCapabilities()
Inicialização da Interface Web
Função JavaScript no painel web que consome /api/capabilities e constrói dinamicamente os botões de câmera, detecta resoluções disponíveis e inicializa a UI para a câmera 0. Roda ao carregar a página.
JavaScript fetch API DOM dinâmico
🔄 updateUIForCamera()
Atualização Dinâmica da UI
Ao trocar de câmera, mostra/esconde cards de controle (ISO, zoom, foco, WB, flash, OIS) conforme as capabilities reais da lente selecionada. Reconstrói botões de modo de foco e white balance dinamicamente.
JavaScript classList.toggle DOM manipulation
📡 RtspServerPedro
Transmissão RTSP H.264
Encapsula o RootEncoder para publicar o stream Camera2 na porta 8554. Parâmetros: resolução 1280×720, 30 FPS, bitrate 2500 kbps, codec H.264. URL de acesso: rtsp://IP:8554/live.
RtspServerPedro.kt RootEncoder 2.4.5 H.264 / RTP
🗂️ CameraCapabilities
Data Class de Capabilities
Data class Kotlin que centraliza todas as informações de uma câmera: ID, hardware level, facing, flags de suporte manual (sensor, post-processing, RAW, burst), ranges numéricos, resoluções, modos AF/AE/AWB, flash, OIS e focais.
CameraCapabilities.kt data class Kotlin

Política de Privacidade

Política de Privacidade — Camera2 RTSP Server
Última atualização: Março de 2026  |  Versão 2.1

🎯 1. Introdução

O aplicativo Camera2 RTSP Server (“o App”), desenvolvido por Luan Silva (“nós” ou “desenvolvedor”), foi criado para transformar dispositivos Android em servidores de streaming de vídeo RTSP para uso em redes locais. Esta Política de Privacidade descreve como o App lida com informações durante o seu uso.

📦 2. Dados Coletados

O App não coleta, armazena, transmite nem compartilha nenhum dado pessoal dos usuários com terceiros. Especificamente:

  • Câmera: acessada exclusivamente para captura e transmissão local do stream de vídeo. Nenhuma imagem ou vídeo é gravado ou enviado a servidores externos.
  • Microfone: permissão declarada para compatibilidade futura com áudio AAC. O App atual não captura áudio.
  • Rede / Wi-Fi: utilizada somente para hospedar os servidores RTSP (porta 8554) e HTTP (porta 8080) na rede local do usuário. Nenhuma conexão é estabelecida com servidores externos.
  • Endereço IP: o IP local do dispositivo é exibido na tela do App para que o usuário possa conectar clientes na mesma rede. Não é transmitido para fora do dispositivo.
  • Dados de diagnóstico: o App não integra Firebase, Analytics, Crashlytics ou qualquer SDK de telemetria.

🔐 3. Permissões Utilizadas

  • android.permission.CAMERA — Necessária para acessar as câmeras do dispositivo e gerar o stream de vídeo.
  • android.permission.RECORD_AUDIO — Declarada para compatibilidade. Não utilizada ativamente na versão atual.
  • android.permission.INTERNET — Necessária para o App funcionar como servidor de rede local (RTSP + HTTP).
  • android.permission.ACCESS_NETWORK_STATE — Utilizada para detectar o estado da conexão de rede.
  • android.permission.ACCESS_WIFI_STATE — Utilizada para obter o endereço IP local do dispositivo na rede Wi-Fi.

🌐 4. Transmissão de Dados

Todo o tráfego de dados é estritamente local (LAN/Wi-Fi). O stream de vídeo é transmitido diretamente do dispositivo Android para os clientes conectados na mesma rede, sem intermediários, proxies ou servidores em nuvem. O App não requer conexão com a internet para funcionar.

👶 5. Uso por Menores

O App não é direcionado a crianças menores de 13 anos. Não coletamos intencionalmente nenhum dado de crianças. Este App destina-se a desenvolvedores, entusiastas de tecnologia e profissionais que utilizam streaming de vídeo em ambientes controlados.

🔗 5. Serviços de Terceiros

O App utiliza as seguintes bibliotecas open-source, que são executadas inteiramente no dispositivo do usuário e não coletam dados:

  • RootEncoder (Pedro SG94) — Licença Apache 2.0. Biblioteca de streaming RTSP/RTMP local.
  • NanoHTTPD — Licença BSD. Servidor HTTP embutido para o painel de controle local.
  • Gson (Google) — Licença Apache 2.0. Serialização de dados JSON local.

✏️ 7. Alterações desta Política

Podemos atualizar esta Política de Privacidade periodicamente. As alterações serão publicadas nesta página com uma nova data de “Última atualização”. O uso continuado do App após alterações constitui aceitação da nova política.
